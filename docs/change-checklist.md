# 变更收尾 Checklist

本清单用于每次功能开发、bug 修复、重构或解析/渲染能力扩展的收尾检查。

## 功能

- [ ] 本次目标功能已经实现
- [ ] 行为变化点已经确认
- [ ] 影响范围已经识别

## 验证

- [ ] 已补充或更新测试
- [ ] 已运行必要的 type-check / unit test / build / 局部验证
- [ ] 如果没有自动化验证，已明确记录人工验证方式

## 文档

- [ ] 已检查 `README.md` 是否需要更新
- [ ] 已检查架构文档、模块说明、计划文档是否需要更新
- [ ] 文档内容与当前代码实现一致

## 知识库

- [ ] 已检查 `docs/knowledge-base/` 是否需要新增记录
- [ ] 已检查现有知识库文档是否需要补充
- [ ] 本次变更中的关键经验、坑点、约束或排查方法已沉淀

## 交付说明

- [ ] 能清楚说明改了什么
- [ ] 能清楚说明为什么这样改
- [ ] 能清楚说明如何验证
- [ ] 能清楚说明新增了哪些文档/知识库

## 使用建议

如果某一项明确“不需要”，也应在任务总结里说明为什么不需要，而不是默认跳过。
